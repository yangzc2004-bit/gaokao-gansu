# 甘肃高考专项计划样例化审查模板

本文件在 `policy-audit-spec.md` 基础上，增加“通过样例 / 拒绝样例 / 缺失信息样例”，用于验证系统审查逻辑是否足够严谨。

说明：
- 通过样例：表示在当前规则引擎下应判定为 `eligible=true`
- 拒绝样例：表示在当前规则引擎下应判定为 `eligible=false`
- 缺失信息样例：表示因关键信息不足，应按从严原则判定为 `eligible=false` 或提示“需补充资料”

---

## 1. 国家专项
- 报考批次：`本科批(C段)`
- 志愿模式：`平行志愿`
- 是否征集志愿：`否`

**通过样例**
- 考生：会宁县户籍，连续 3 年
- 学籍：会宁县高中连续 3 年，且实际就读
- 监护人：会宁县户籍连续 3 年
- 民族：汉族
- 专项失信：无
- 结果：通过

**拒绝样例**
- 考生：会宁县户籍连续 3 年
- 学籍：兰州市城关区高中
- 监护人：会宁县户籍
- 结果：拒绝，原因是学籍不在实施区域，三年统一不成立

**缺失信息样例**
- 考生：会宁县户籍
- 学籍：会宁县高中
- 监护人：未提供户籍信息
- 结果：从严拒绝，原因是监护人实施区域信息缺失

- 报考批次：`提前批A段之前单独投档`
- 志愿模式：`单独报名+校测`

- 结果：拒绝

**缺失信息样例**
- 只说“想报强基”，未提供审核状态
- 结果：从严拒绝

## 使用建议

你接下来审核这份文档时，重点看三类计划：
- **高风险误判类**：`高校专项`、`两州一县专项`、`强基计划`、`综合评价录取`
- **地区边界类**：`国家专项`、`革命老区专项`、`其他民族地区专项`
- **身份强约束类**：`建档立卡专项`、`边防军人子女预科`、`少数民族预科班`
