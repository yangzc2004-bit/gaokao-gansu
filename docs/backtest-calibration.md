# 概率模块 V2 回测校准

## 回测口径
- 只使用真实历史数据
- 使用 `metric_slot=2/3` 作为历史信息
- 使用 `metric_slot=1` 作为留出的真实结果
- 不补造缺失年份数据
- 用历史信息预测最新一年，再和最新一年真实最低位次对比

## 样本构造
- 回测记录数：`22262`
- 回测样本数：`155834`
- 候选位次通过相对真实最低位次的模拟倍率生成：
  - `0.85`
  - `0.90`
  - `0.95`
  - `1.00`
  - `1.05`
  - `1.10`
  - `1.15`

判定规则：
- 若 `user_rank <= 最新一年真实最低位次`，记为真实可录取
- 否则记为真实未录取

## 当前阈值
- `冲 >= 0.25`
- `稳 >= 0.50`
- `保 >= 0.70`

## 回测结果

### 按风险桶看（当前阈值下）
- `冲`：样本 `31700`，真实录取率 `41.32%`，平均预测概率 `0.300`
- `稳`：样本 `19461`，真实录取率 `70.16%`，平均预测概率 `0.516`
- `保`：样本 `15016`，真实录取率 `90.46%`，平均预测概率 `0.768`

### 按概率区间看
- `(0.0, 0.2]`：真实录取率 `27.48%`
- `(0.2, 0.4]`：真实录取率 `43.30%`
- `(0.4, 0.6]`：真实录取率 `69.03%`
- `(0.6, 0.8]`：真实录取率 `85.39%`
- `(0.8, 1.0]`：真实录取率 `90.70%`

### 按历史年数看
- `1年历史`：平均预测概率 `0.349`，推荐率 `19.47%`
- `2年历史`：平均预测概率 `0.420`，推荐率 `48.37%`

## 阈值扫描结论

### 冲
- `0.20` → 录取率 `41.32%`
- `0.25` → 录取率 `44.30%`
- `0.30` → 录取率 `46.97%`

结论：
- 若想保留更多冲刺选择，`0.20` 合理
- 若想更保守，可上调到 `0.25`

### 稳
- `0.45` → 录取率 `70.16%`
- `0.50` → 录取率 `74.30%`
- `0.55` → 录取率 `78.36%`

结论：
- `0.45` 已经能把“稳”维持在约 7 成真实录取率
- 若想进一步收紧，可提升到 `0.50`

### 保
- `0.70` → 录取率 `90.46%`
- `0.75` → 录取率 `91.34%`
- `0.80` → 录取率 `91.39%`

结论：
- `0.70` 已经足够保守
- 再往上提，提升有限，但会明显减少结果数

## 当前建议
当前阈值：
- `冲 >= 0.20`
- `稳 >= 0.45`
- `保 >= 0.70`

是可以接受的一版默认参数。

如果产品倾向更保守，可以改成：
- `冲 >= 0.25`
- `稳 >= 0.50`
- `保 >= 0.70`
